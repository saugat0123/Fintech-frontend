User Documentation First step

Step #1 : Create Branch

Click on Branch Menu Click on . . Branch Page Go to create Branch page Second step

Step #2 : Create Role and assign permissions to roles

Click on Role And Permission Menu Click on Role And Permission Page Rules :
  1. Role Name is unique .
  2. Role Type :
    1. There will be only one MAKER type Role .
    2. Until Maker is not selected in Role Type ,Role Type option is visible
    3. After any role is assigned as MAKER then rest of the inserted role will be of APPROVAL TYPE.
  3. Role ACCESS
    1. Role ACCESS gives the authority for role to access data of BRANCH. Branch is assign to user which will be discuss in create User.
      1. SPECIFIC
        • Role assign as SPECIFIC can access data from multiple branch (for eg . REGIONAL MANAGER).
      2. OWN
        • Role assign as OWN can access data from only assigned branch (for eg . RELATION MANAGER).
      3. ALL
        • Role assign as ALL can access data from all branch .
Go to create Role page

Step #3 : Assign Role Hierarchy

Click on Role Hierarchy Menu Double click and drag up/down to change the role hierarchy. Rule
  1. Role hierarchy defines the authority level of the role.
  2. Newly added role will appear in the last in the hierarchy. So it is very important to set role hierarchy.
Go to Role Hierarchy page Fourth step

Step content #4 : Assign Role & Permission

Click on Role And Permission Menu click on and then a role you want to assign permissions. Go to Role and Permission page Fifth step

Step #5 : Create User

Click on Users Menu Click on to add user Rules:
  1. A role can be assigned to only one user on a branch. For example if role type of manager is already assigned on branch "abc" to a user, it can not be assigned to new user on same branch.
  2. A role of type specific can be assigned to multiple branches.For example "Regional manager"
  3. A role of type All is assigned to all branches so select branches option is not needed. For example "CEO".
  4. A role of type own can be assigned to a single branch. For example "Branch Manager"
  5. user can be dismissed by clicking on .
  6. By dismissing a user a branch and role is also dismissed.
Go to Create User page
